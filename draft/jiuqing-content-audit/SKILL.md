# 发布前内容质检

解决"发布后才发现问题"的痛点。执行 5 维检查，输出红黄绿分级报告。

**约束**：只标注不修改原文（修改交 text-humanize 或用户自行处理）；不联网验证事实（仅标注可疑项供确认）；默认公众号标准，用户可指定平台；效果承诺禁止——不保证"发布后零问题"。

---

## 1. 完整性

扫描并标记：
- **占位符**：`TODO` `FIXME` `TBD` `待补充` `待确认` `xxx` `【请填写】` `{{}}` → 🔴
- **空白段落**：连续 2 行以上无有效内容 → 🟡
- **可疑链接**：域名含 `example` `test` `localhost` `placeholder` 或格式明显不完整（无协议、路径截断）→ 🟡 提示"请人工确认有效性"

**完成标志**：输出完整性检查结果，每项标注位置（P段落号-L行号）和级别。

## 2. 事实性

标注可疑项，不联网验证：
- **日期矛盾**：未来日期（超出合理范围）、相对时间无锚点（"上周""去年"缺基准）→ 🟡
- **数字缺单位**：纯数字 ≥ 100 无量词（`万/亿/元/人/%/天`）→ 🟡
- **命名不一致**：同一实体出现 ≥ 2 种写法（如 `iPhone 15` / `iphone15`、`ChatGPT` / `Chat GPT`、`字节跳动` / `字节`）→ 🔴

## 3. 可读性

手机端阅读体验标准：
- **段落过长**：单段 > 150 字 → 🟡；> 300 字 → 🔴
- **句子过长**：单句 > 40 字（含标点）→ 🔴；> 30 字 → 🟡
- **列表密度**：连续列表项 > 7 项无段落间隔 → 🟡
- **小标题间隔**：正文连续 > 400 字无小标题 → 🟡

## 4. 风格一致性 — AI 套话检测

标记以下关键词出现处（可扩展），建议替换为更直接的表达：

| 类型 | 关键词清单 |
|------|-----------|
| 开头/过渡 | `值得注意的是` `在当今社会` `在当今时代` `众所周知` `毋庸置疑` `不难发现` |
| 总结/收尾 | `综上所述` `总而言之` `总的来说` `简而言之` `从某种意义上说` |
| 泛化表达 | `随着…的发展` `引起了广泛关注` `越来越受到` `不可或缺` `至关重要` `起到了重要作用` |
| 营销套话 | `赋能` `抓手` `闭环` `颗粒度` `打法` `势能`（非技术语境时） |

出现即 → 🟡

## 5. 平台适配（默认公众号）

| 平台 | 标题长度 | 标签格式 | 备注 |
|------|---------|---------|------|
| 公众号 | ≤ 64 字，> 64 → 🔴；48-64 → 🟡 | `#话题#` | 默认标准 |
| 小红书 | ≤ 20 字 | `#话题` | 吸引力优先 |
| 知乎 | ≤ 50 字 | 无 `#` | 准确性优先 |
| 抖音/微博 | ≤ 30 字 | `#话题` | 简洁有力 |

- **违禁词扫描**：读取外部配置（如 `config/banned-words-{platform}.txt`），留配置入口不硬编码。命中 → 🔴，无配置文件则跳过。

**完成标志（5 维扫描全部完成）**：所有 5 维扫描结束，按输出格式输出完整质检报告（含红黄绿分级、位置标注、修改建议）。

---

## 输出格式

```
=== 内容质检报告 ===
平台：[平台] | 字数：[N] | 检查时间：[日期]

🔴 红牌（必须修改）X 项
[P3-L18] 事实性：数字 "5000" 无单位 → 补充"5000元"或"5000人"
[P7-L3] 完整性：发现占位符 "TODO: 补充数据" → 删除或替换

🟡 黄牌（建议优化）X 项
[P2-L1] 可读性：段落 248 字，超阈值 → 拆分为 2 段
[P5-L8] 风格：套话"值得注意的是" → 删除或改为直接陈述

🟢 绿牌（通过）X 项
- 无占位符  |  日期格式统一  |  命名一致  |  无违禁词

=== 汇总：🔴 X 项 | 🟡 X 项 | 🟢 X 项 ===
```

每条标注包含：级别 + 维度 + 位置（段落号 P / 行号 L）+ 问题描述 + 修改建议。

---

## 配置扩展

- **AI 套话清单**：用户可追加关键词，如"检查时加上：赋能、抓手"
- **违禁词库**：按平台提供外部文件，首次使用提示用户配置
- **阈值调整**：用户一句话即可覆盖默认值，如"用小红书标准"或"段落阈值改为 100 字"
- **平台切换**：指定平台名即自动适配标题长度、标签格式等规则

## 使用示例

> "用小红书标准检查这篇文章" → 切换平台阈值，执行 5 维扫描，输出报告。
> "检查时加上违禁词列表：config/banned-words-xhs.txt" → 加载自定义词库。

## 失败路径表

| 情况 | 信号 | 处理 |
|------|------|------|
| 用户要求修改原文 | 出现"帮我改""直接修""修改这些" | 拒绝修改——本技能只标注不修改，引导使用 text-humanize 或 article-edit |
| 违禁词配置文件不存在 | 读取 config/banned-words-{platform}.txt 失败 | 跳过违禁词扫描，在报告中注明"未找到违禁词配置文件，已跳过此项检查" |
| 文章过短（<100字） | 输入字数不足以覆盖5维检查 | 仍执行完整5维检查，在报告中注明篇幅限制，多数维度可能标记为🟢通过 |
| 用户要求验证事实准确性 | 出现"核实""验证""查证" | 拒绝——本技能不联网验证事实，仅标注可疑项供用户确认 |
| 用户要求超出平台范围的检查 | 如"帮我检查SEO""检查无障碍" | 告知超出5维检查范围，建议使用对应专项工具 |
| 5维全部通过无问题 | 报告中🔴和🟡均为0 | 如实输出全绿报告，不凑问题——确实没问题就报告没问题 |
