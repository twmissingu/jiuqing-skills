# jiuqing-crossplatform-convert：跨平台内容格式适配

## 工作流

1. **调用 idea-grill 对齐**：目标平台列表 + 各平台调性 + 发布时间要求

**完成标志（对齐）**：确认目标平台数量、各平台调性偏好、字数/格式要求。

2. **输入输出**：输入一篇完整文章（或素材包：标题+正文+图片+数据），输出每个平台的适配版本，统一在一个输出中展示

3. **平台适配规则**（每个平台独立处理）：

   | 平台 | 格式要求 |
   |------|----------|
   | **公众号** | HTML 排版（`<h2>`/`<p>`/`<ul>`/`<blockquote>`），封面 900×383，摘要 ≤120 字，原文链接 |
   | **小红书** | emoji 分段（每段 ≤100 字），标题 ≤20 字，标签 3-5 个，对话语气 |
   | **知乎** | Markdown 结构化，首段 200 字摘要，数据引用标注，分割线 |
   | **抖音/视频号** | 口播脚本格式（开场 hook → 展开 → 互动引导），时长 30-60 秒 |
   | **B站** | 醒目标题 + 分段 + 表情包标记 + 时间戳 |

4. **每个版本提供**：适配版本全文 + 适配说明（改了哪些地方、为什么）

5. **标签/话题建议**（平台不同策略不同）

6. **核心信息一致性检查**：检查所有版本是否保留了原文的核心观点、关键数据和结论

**完成标志（全部完成）**：输出每个平台的适配版本全文 + 适配说明 + 标签建议，最后一个版本输出后附加核心信息一致性检查结论（保留/遗漏了哪些核心信息）。

## 与 article-rewrite 的区别

- **article-rewrite**：深度改写（大幅压缩、改变叙事节奏）
- **本 skill**：结构适配（保留信息密度，调整格式和语态）
- **建议配合使用**：先用 article-rewrite 做深度压缩，再用本 skill 做平台适配

## 边界

- 只产出文本，不发布
- 不做跨语言翻译
- 不做深度内容改写（那是 `article-rewrite` 的活）
- 效果承诺禁止——不保证"发布后效果好"或"阅读量提升"
- 动态/视频内容禁止——只输出文本格式适配，不生成视频脚本以外的动态内容

## 失败路径表

| 情况 | 信号 | 处理 |
|------|------|------|
| 用户未指定目标平台 | 只说"适配多平台"但没说哪些 | 调用 idea-grill 确认目标平台列表，不自行猜测 |
| 原文过短（<200字） | 输入字数不足 | 提醒字数不足以覆盖多平台格式要求，建议补充内容或只适配部分平台 |
| 跨语言翻译需求 | 出现"转成英文""翻译" | 拒绝——本技能只做格式适配不做语言转换 |
| 用户要求自动发布 | 出现"直接发""帮我发布" | 拒绝自动发布，只产出文本供用户审阅 |
| idea-grill 对齐失败 | 依赖调用超时或报错 | 降级处理：使用各平台默认规则（公众号HTML/小红书emoji/知乎MD/抖音口播）继续适配 |
| 目标平台不在已知列表中 | 如"适配LinkedIn版" | 按通用原则（字数+结构+调性）适配，注明无专属模板 |

## 示例：短文 → 公众号版 + 小红书版

### 原文

> 2024 年中国新能源汽车销量达 1,286 万辆，同比增长 35.5%，渗透率突破 40%。比亚迪以 425 万辆稳居第一，市占率 33%。宁德时代全球动力电池装机量 259.7 GWh，连续 8 年第一。行业预计 2025 年渗透率将达 50%。

### 公众号版

```html
<h2>2024 年中国新能源汽车销量突破 1,286 万辆，渗透率首超 40%</h2>
<p>2024 年是中国新能源汽车产业里程碑式的一年。全年销量达到 <strong>1,286 万辆</strong>，同比增长 35.5%，市场渗透率首次突破 <strong>40%</strong> 大关。</p>
<blockquote>
<p>核心数据一览</p>
<ul>
<li>总销量：1,286 万辆（+35.5%）</li>
<li>渗透率：40%+</li>
<li>比亚迪销量：425 万辆，市占率 33%</li>
<li>宁德时代装机量：259.7 GWh，全球 8 连冠</li>
</ul>
</blockquote>
<p>行业预测，2025 年渗透率有望达到 50%，新能源取代燃油车已进入快车道。</p>
```
**适配说明**：改为 HTML 结构，用 `<blockquote>` + `<ul>` 突出数据，`<strong>` 强调关键数字，适合公众号读者快速扫读。
