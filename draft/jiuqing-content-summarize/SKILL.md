# 长文摘要

解决"长文消化慢、读完只记住大概"的痛点。将长文压缩为结构化、可引用的卡片。

## 输入方式

接受三种输入，任选其一：
- **粘贴原文**：直接发送文本
- **URL**：提取正文后摘要
- **文件路径**：读取本地文件

输入超 15000 字时，提醒用户建议拆分。

## 模式一：单篇摘要

输出结构：
- 一句话核心结论（≤30 字）
- 3-5 个关键论点（每个附 `【段落N】` 出处标记）
- 可直接引用的精炼段落（标 `✅可直接用` 或 `⚠️需改写`）
- 潜在争议点/待验证信息（标 `❓`）

**完成标志**：输出含一句话结论 + 3-5个带出处的关键论点 + 可引用段落 + 待验证项的完整卡片。

**示例输入：** 一篇 5000 字的 AI 编程工具评测文（粘贴全文）

**示例输出片段：**
```
## 一句话结论
Cursor 在代码生成准确率上领先，但长任务稳定性不如 Claude Code。

## 关键论点
1. **Cursor 代码生成准确率最高** — 【段落4-6】
   > "50 个基准测试中，Cursor 通过率 78%，Claude Code 71%"
2. **长任务上下文易丢失** — 【段落12】
   > "超过 20 轮对话后，Cursor 开始遗忘早期约束"

## 可引用段落
- ✅ 可直接用："Cursor 在准确率上领先，但长任务稳定性不足"
- ⚠️ 需改写："具体基准测试数据见原文表格 2"（需补充表格编号）

## 待验证
- ❓ 基准测试用例是否覆盖真实业务场景
```

## 模式二：多篇对比摘要

输入 2-5 篇，输出：

**示例输入：** 文章A《开源AI的优势》、文章B《商业AI的壁垒》

**完成标志**：输出含共识点/矛盾点/互补点的对比分析，每点附原文出处标记。

**示例输出片段：**
```
## 共识点
- 均认为数据质量是 AI 落地的关键【A-段落3】【B-段落7】

## 矛盾点
- A 主张开源协作加速创新；B 强调知识产权保护才能持续投入

## 互补点
- A 提供技术实现细节，B 补充商业模式和合规路径
```

## 模式三：渐进式摘要

分层展开：
1. 先给一句话核心结论
2. 用户回复想展开的编号
3. 展开该点的子论点与出处
4. 用户可继续追问，逐层给出原文引用

**完成标志（第1层）**：输出一句话核心结论；后续层级待用户指令再展开。

## 输出规则

- 统一 Markdown 格式，结构清晰
- 每条论点必须附原文出处（段落号或引用句）
- 保持原文语气和立场，只压缩不扭曲
- 输入什么语言就输出什么语言，不做翻译
- 仅标记可疑信息（`❓`），不主动搜索核实
- 不生成原文不存在的观点
- 效果承诺禁止——不保证"完全覆盖原文要点"或"零遗漏"

## 行为准则

- 默认使用单篇摘要
- 若输入多篇未指定模式，询问是否需要对比分析
- 渐进式模式下，等待用户指令再逐层展开

## 失败路径表

| 情况 | 信号 | 处理 |
|------|------|------|
| 输入超15000字 | 字数统计超限 | 提醒用户建议拆分，但仍尝试摘要核心内容；若无法处理则建议分段输入 |
| 用户要求核实数据准确性 | 出现"核实""验证""确认数据""查一下" | 拒绝——本技能仅标记可疑信息（❓），不联网验证事实 |
| 用户要求添加自己的分析或建议 | 出现"你的看法""你觉得""分析一下""给建议" | 拒绝生成原文不存在的观点，只做压缩整理 |
| 渐进式模式下用户一次展开多项 | 用户回复"全部展开"或编号列表 | 一次性展开所有子论点，但保持结构清晰，不混淆层级 |
| 多篇对比但文章数量>5篇 | 输入超过5篇 | 告知对比模式最多支持5篇，建议用户筛选最重要的5篇 |
| 内容为非文本格式（图片/PDF无法读取） | 文件格式不支持 | 告知仅支持纯文本/URL/本地文件，不支持图片内文字提取 |
