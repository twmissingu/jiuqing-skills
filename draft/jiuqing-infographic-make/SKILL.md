# jiuqing-infographic-make：文字转信息图

解决"文字转信息图没有框架"的痛点。参考 baoyu-infographic（21 种布局×22 风格）设计方法，但精简适配 jiuqing 仓库风格。

## 输入输出

**输入**：文章 / 主题 / 数据点  
**输出**：一段经过 jiuqing-prompt-refine 打磨后的文生图提示词

## 8 种常用信息图布局

| 布局 | 适用场景 |
|------|---------|
| **时间线** | 历史演进、路线图、里程碑 |
| **对比卡** | 两列 PK、优缺点、A/B 对比 |
| **步骤流程** | 操作指南、1-2-3-4 步骤 |
| **列表清单** | Top N、Checklist、要点汇总 |
| **金字塔/层次** | 主次分层、优先级、能力模型 |
| **数据统计** | 数字 + 解释、核心指标 |
| **剖面/解剖** | 实体 + 标注、产品拆解 |
| **引语金句** | 大字号引用 + 署名 + 背景 |

## 生成流程

1. **分析内容** → 提取核心信息与结构。完成标志：输出信息点列表 + 结构类型（并列/递进/对比/层级）。
2. **选布局** → 在 8 种布局中逐一匹配：有时间维度→时间线，有对比→对比卡，有步骤→步骤流程，有层级→金字塔，有数字→数据统计，有引语→金句，多条目→列表清单，有拆解→剖面。
3. **抽关键信息点** → 每块 ≤3 个要点，总信息块 ≤8。超限时合并次要项或拆分为多张图。
4. **结构化布局描述** → 400-600 字自然语言描述每块位置、内容、配色（400字覆盖核心布局，600字追加装饰细节——低于400信息不足，高于600图像生成模型易忽略后半段）。
5. **调用 jiuqing-prompt-refine 打磨** prompt。prompt-refine 不可用时跳过打磨直接输出。
6. **输出**给 image-generate 执行。

> 关键约束：在生成最终 prompt **之前**必须先调用 jiuqing-prompt-refine。

## 布局描述结构

用自然语言描述，确保 image-generate 能准确还原：

- **整体构图**：方向（横向/纵向）、页面比例、分区方式
- **标题区**：位置、字号层级、主副标题关系
- **内容块**：每块的位置、形状、内容要点、箭头/连线逻辑
- **配色**：指定色板中的颜色用途（背景、标题、高亮、正文）
- **装饰**：图标/分隔线/圆角/阴影的使用

字数控制在 400-600 字，忌过于简短导致歧义。

## 配色方案推荐（5 组）

1. **极简白** `#FFFFFF` `#F5F5F5` `#333333` `#007AFF` — 适合严肃内容
2. **暖日落** `#FF6B35` `#F7C59F` `#2E294E` `#FBF5F3` — 适合生活/创意
3. **森林** `#2D6A4F` `#95D5B2` `#1B4332` `#F0FFF0` — 适合环保/自然
4. **科技蓝** `#0A192F` `#172A45` `#64FFDA` `#E6F1FF` — 适合技术/数据
5. **莫兰迪** `#B8A9C9` `#E8D5C4` `#7C6F64` `#F9F6F0` — 适合人文/品牌

## 边界

- **不直接出图**：打磨后交由 image-generate 执行
- **不包含动态图表**（动画 / GIF）
- **不包含复杂数据可视化**（柱状图/折线图用 matplotlib 更合适）
- 不做视频/幻灯片（仅静态信息图）

## 失败路径

| 情况 | 信号 | 处理 |
|------|------|------|
| 输入缺少内容 | 用户只说"做个信息图"无内容 | 追问："请提供文章内容、主题或数据点" |
| 用户要求直接出图 | "帮我生成PNG/JPEG" | 拒绝并指引："本 skill 生成信息图 prompt，出图由 image-generate 执行" |
| 信息点过多 | 输入含 >10 个要点 | 合并次要项（每组 ≤3 要点，总块 ≤8），仍超则建议拆分为多张图 |
| 用户要求动态/视频 | "做成动图""要动画效果" | 拒绝："本 skill 只生成静态信息图 prompt，动画效果请使用 CSS animation 或专用工具" |
| prompt-refine 返回失败 | 超时或异常 | 跳过打磨直接输出原始 prompt，标注"未经 prompt-refine 打磨" |

## 示例："对比 Cursor 和 Claude Code"

### 分析内容
两个 AI 编程工具的差异：Cursor 强调 IDE 集成与可视化，Claude Code 强调终端原生与深度推理。

### 选布局
**对比卡**（两列 PK）

### 结构化的布局描述（约 450 字）

```
纵向 2:3 信息卡片。背景用极简白方案（#FFFFFF），上方居中标题区：大号粗体
"Cursor vs Claude Code"（#333333），副标题"AI 编程工具深度对比"
（#666666，14px）。主体为左右对称两列，中间用 2px 竖线（#E0E0E0）分隔。

左列（Cursor，#007AFF 为强调色）：顶部圆形头像 + 标识"Cursor"，
下方三个要点——①"IDE 原生集成"②"可视化 Diff"③"多文件编辑"，
每点前有蓝色圆点标记。底部标签"适合：视觉型开发者"。

右列（Claude Code，#2E294E 为强调色）：顶部圆形头像 + 标识"Claude Code"，
下方三个要点——①"终端原生体验"②"深度推理链路"③"Git 流程嵌入"，
每点前有紫色圆点标记。底部标签"适合：终端型开发者"。

底部横跨两列的总结栏：浅灰背景（#F5F5F5），灰色字体
"结论：场景互补，非替代关系"，左右各一个箭头指向中间。
```
