# jiuqing-newsletter-curate — 资讯策展技能

## 1. 对齐策展范围

调用 `jiuqing-idea-grill` 对齐以下维度：
- **行业/主题**（如 AI 编程工具、跨境出海、Web3）
- **受众画像**（开发者 / 产品经理 / 泛科技读者）
- **格式偏好**（长版 / 图文版 / 简报版）
- **频率**（日报 / 周报 / 双周报）

## 2. 策展流程

### Step 1：多渠道采集（候选池 20-30 条）
- `web_search` 搜索行业热点关键词
- 指定 RSS / 站点提取（TechCrunch、ArsTechnica、HackerNews、GitHub Trending 等）
- 参考 `kangaroo-viral-topic` 路由：按平台/信源打分排序去重

**完成标志**：采集到 20-30 条候选条目，每条含标题、来源、链接、一句话摘要。

### Step 2：筛选 5-8 条高价值条目
每条结构化输出：

```
标题 | 来源 | 链接
一句话摘要（≤30 字）
个人点评角度（为什么这条值得关注）
价值标记：[🔥行业风向 / 💡实操指南 / 📊数据报告 / 📰行业动态]
```

**完成标志**：输出 5-8 条结构化条目，每条含标题/来源/链接/摘要/点评/价值标记。

### Step 3：编辑引言（150-250 字）
串联本期主题，点明"为什么这期值得读"，有观点有态度。

**完成标志**：输出 150-250 字引言，串联本期主题并给出编辑态度。

### Step 4：趋势观察
提炼本期内容间的共性信号、矛盾点或值得警惕的趋势。

**完成标志**：输出 2-3 条趋势观察，每条串联多条内容得出信号。

### Step 5：格式适配
按对齐结果输出对应格式。

**完成标志**：输出按目标格式排版的完整 newsletter 内容。

## 3. 三种输出格式

### 长版（公众号）
引言 + 5-8 条详情 + 趋势观察，每条 200-300 字，适合深度读者。

### 图文版（小红书/微博）
核心 3-5 条，emoji 分段，配图建议（信息图 / 截图 / meme），适合碎片化阅读。

### 简报版（简洁）
5-8 条标题 + 一句话摘要，可快速扫读，适合日报场景。

## 4. 风格要求

- **有观点有态度**：不是资讯搬运工，每条必须带个人点评
- **口语但不失信息量**：像资深编辑在跟你聊天
- **敢于下判断**：这条是忽悠还是真机会，直接说

## 5. 采编频率

尊重用户设定的频率，不主动加量。用户说"每天做"就每天，用户说"周报"就每周。

## 边界

| 边界 | 说明 |
|------|------|
| ❌ 不做深入调研 | 那是 `research-structured` 的活 |
| ❌ 不生成原创长文 | 策展 ≠ 创作 |
| ❌ 不自动发布 | 输出供用户审阅后再发布 |
| ❌ 不强行凑数 | 优质条目 <5 则时，宁可减量 |
| ❌ 效果承诺禁止 | 不承诺"阅读量提升"或"关注增长" |
| ❌ 不做事实核查 | 仅标注信息来源，不验证准确性 |

## 失败路径表

| 情况 | 信号 | 处理 |
|------|------|------|
| 候选池不足20条 | web_search 和 RSS 返回结果稀少 | 如实报告采集数量，在已有候选中筛选高价值条目；宁可减量不凑数 |
| 优质条目不足5条 | 筛选后高价值条目<5 | 如实输出已有条目，在引言中注明本期精选较少，不为凑数降低筛选标准 |
| 用户要求深入分析某条新闻 | 出现"展开讲讲""深入分析""技术原理" | 完成策展后提示使用 research-structured 做深入分析，不在此技能内展开 |
| 用户要求自动发布 | 出现"直接发""帮我发布""推送" | 拒绝自动发布，输出供用户审阅后再发布 |
| idea-grill 对齐失败 | 依赖调用超时或报错 | 降级处理：从用户原始输入提取行业/主题/格式，直接进入 Step 1 采集 |
| 用户要求原创内容而非策展 | 出现"写一篇文章""原创""撰稿" | 拒绝——策展是整理和点评已有资讯，不做原创长文创作 |

## 示例：AI 编程工具周报（长版片段）

**编辑引言**

> 这周的 AI 编程工具圈有点意思——Anthropic 悄悄把 Claude 的代码生成能力又推了一层，Cursor 的聊天式重构让老项目也能"边聊边改"，而 GitHub Copilot 的 Agent Mode 终于不再只是个 autocomplete。三家各走一路：一个拼理解深度，一个拼交互体验，一个拼生态集成。本周 6 条精选，帮你理清格局。

### 🔥 Cursor 推出 Chat-based Refactoring，老项目改造门槛大降
**来源**：Cursor Blog | [链接](#)

一句话摘要：在聊天窗口里选中代码就能重构，无需手动 diff。
**点评**：这是 Cursor 差异化最明显的一步——让重构从"写 prompt→等 diff→手动合并"变成自然对话。对维护老项目的团队是实打实的效率提升。

### 💡 Claude 代码理解能力再升级，支持超长上下文切片分析
**来源**：Anthropic Release Notes | [链接](#)

一句话摘要：可一次性分析 5 万+ token 的代码仓库。
**点评**：Anthropic 在代码理解上的投入明显转向"深度"而非"宽度"。这对于大型 monorepo 场景是福音，但也意味着更贵的 API 账单。

### 📰 GitHub Copilot Agent Mode 正式开放，CI/CD 全链路可调
**来源**：GitHub Blog | [链接](#)

一句话摘要：Copilot 现在能在终端里跑测试、查日志、修 bug。
**点评**：从"帮你写代码"到"帮你跑代码"，Copilot 正在蚕食传统 DevOps 工具的地盘。值得关注的是它如何处理权限和安全边界。

### 📊 Stack Overflow 2026 开发者调查：AI 工具使用率突破 82%
**来源**：Stack Overflow | [链接](#)

一句话摘要：82% 开发者日常使用 AI 编程工具，比去年上升 15 个百分点。
**点评**：数据不意外，但值得关注的是"信任度"曲线——多数开发者仍会手动 review AI 生成代码，说明工具还有很大提升空间。

### 💡 开源项目 aider 登顶 GitHub Trending，纯终端 AI 结对编程
**来源**：GitHub | [链接](#)

一句话摘要：不开 IDE，终端里就能和 AI 结对编程。
**点评**：这波热度说明 CLI 派开发者并不小众。aider 的优势在于轻量和 git-aware——自动 commit、自动 diff，对习惯终端工作流的开发者极其友好。

### 🔥 Replit Agent 支持自定义模型接入，不再绑定自家模型
**来源**：Replit Blog | [链接](#)

一句话摘要：用户可接入 Claude / GPT / 自部署模型。
**点评**：Replit 这一步很聪明——与其卷模型不如做平台。但多模型调度的一致性和调试体验是新的挑战。

**趋势观察**

本周几个信号值得串起来看：
1. **从"写代码"到"管代码"**：Copilot Agent Mode 和 Cursor 重构能力指向同一个方向——AI 不再只是输入工具，而是参与软件全生命周期。
2. **开源 vs 封闭的路径分化**：aider + Replit 的开源/开放路线和 Claude/Copilot 的封闭路线形成鲜明对比，未来可能会出现"工具碎片化"的新问题。
3. **开发者信任仍是瓶颈**：Stack Overflow 的数据很诚实——82% 使用率 ≠ 82% 信任度。下一阶段的竞争可能在"可解释性"和"可审计性"上。
