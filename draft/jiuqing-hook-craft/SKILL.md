## 工作流

### 1. 调用 idea-grill 对齐
调用 `jiuqing-idea-grill` 获取目标平台、读者群体、内容类型。
完成标志：拿到平台+受众+内容类型三要素，缺任一则追问用户。
idea-grill 不可用时：直接用用户原始输入生成，标注"未经对齐"。

### 2. 生成 5-8 个开头钩子（5 个覆盖核心角度已够决策，8 个确保多样性——少于 5 选择不足，多于 8 决策疲劳）
基于输入内容摘要/主题，从以下 8 种角度输出开头钩子（前 1-3 句）：

| 切入角度 | 模板示例 |
|---|---|
| 提问式 | "你有没有想过，为什么XX…" |
| 数据式 | "XX%的人不知道，…" |
| 反直觉式 | "你以为XX很安全？错了" |
| 故事式 | "上周我差点踩了一个大坑…" |
| 痛点直击式 | "你是不是也遇到过…" |
| 金句宣言式 | "XX，是性价比最高的投资" |
| 场景代入式 | "想象一下：你刚打开XX，…" |
| 引用权威式 | "XX曾说过…" |

每个候选附：
- 切入角度（以上 8 种之一）
- 适用内容类型（评测/教程/观点/故事）
- 预估效果（冲击力 1-5 + 自然度 1-5）
- 适合平台标记（公众号/小红书/抖音/B站/知乎）

### 3. 推荐 Top 3
从候选钩子中推荐最优的 3 个，并说明理由。

## 边界规则
- 只生成前 1-3 句开头，不是整篇文章
- 不做标题（title-craft 负责）
- 不改已有开头内容（article-edit 负责）
- 不涉及 SEO 关键词策略
- 不承诺效果（"10万+""爆款"等），钩子质量取决于内容匹配度和平台算法

## 失败路径

| 情况 | 信号 | 处理 |
|------|------|------|
| 输入缺少主题/内容 | 用户只说"写个开头"无具体内容 | 追问："请提供文章主题或内容概要，以及目标平台和读者群体" |
| 用户要求写标题 | 输入含"标题""起名"等 | 拒绝并指引："标题生成请使用 title-craft，本 skill 只负责开头钩子" |
| 用户要求写完整文章 | 输入含"完整文章""2000字"等 | 拒绝并聚焦："本 skill 只生成 1-3 句开头钩子，完整文章请直接描述需求让 agent 撰写" |
| idea-grill 返回信息不足 | 超时或返回模糊/空结果 | 跳过对齐，直接用用户原始输入生成，标注"未经对齐，基于原始输入" |
| 用户否定全部候选 | "都不好""再来一批" | 分析否定原因（太长/太硬/不自然），调整角度权重后重新生成，最多 3 轮 |
| 主题过窄无法填满 8 角度 | 如"某品牌某型号参数对比" | 减少到 3-5 个最匹配角度，不必强行凑满 8 种 |

## 示例：Cursor vs Claude Code

**输入：** 评测对比 Cursor 和 Claude Code 两款 AI 编程工具，目标读者是开发者，发在 B站 和知乎。

### 候选开头

**① 数据式**
> "70% 的开发者还在手动写 CRUD——而用 Cursor 和 Claude Code 的人，已经把效率提升了 3 倍以上。"
- 切入角度：数据式
- 适用类型：评测
- 预估效果：冲击力 4 · 自然度 3
- 适合平台：B站 / 知乎

**② 提问式**
> "你有没有想过，为什么同样做 AI 编程，有人选 Cursor，有人却死磕 Claude Code？"
- 切入角度：提问式
- 适用类型：教程
- 预估效果：冲击力 3 · 自然度 5
- 适合平台：知乎 / B站 / 公众号

**③ 反直觉式**
> "你以为 AI 写代码已经够强了？我用 Cursor 和 Claude Code 分别做了一个完整项目，结果让我震惊。"
- 切入角度：反直觉式
- 适用类型：评测
- 预估效果：冲击力 5 · 自然度 4
- 适合平台：B站 / 抖音

**④ 故事式**
> "上周我接了个紧急需求，3 天工期，我同时打开了 Cursor 和 Claude Code——一个帮我写了后端，一个搞定了前端。"
- 切入角度：故事式
- 适用类型：故事
- 预估效果：冲击力 4 · 自然度 5
- 适合平台：B站 / 小红书 / 抖音

**⑤ 痛点直击式**
> "你是不是也遇到过：需求改了三版、 Deadline 越来越近，但代码才刚开了个头？"
- 切入角度：痛点直击式
- 适用类型：教程
- 预估效果：冲击力 4 · 自然度 4
- 适合平台：知乎 / 公众号 / 抖音

### Top 3 推荐

| 排名 | 钩子 | 理由 |
|---|---|---|
| 🥇 | ③ 反直觉式 | 冲击力最强（5），直接打破观众"AI 写代码已够强"的预设，悬疑感驱动完播；评测类首发适合用反差抓注意力 |
| 🥈 | ④ 故事式 | 自然度最高（5），以真实经历代入，让技术话题有人情味；B站和小红书算法偏好叙事开头 |
| 🥉 | ② 提问式 | 平衡了冲击力和自然度（3+5），开放式问题引导观众在脑中形成立场，适合知乎深度讨论和公众号转化 |
