## 封面图设计技能

**痛点**：封面图缺乏系统控制，每次依赖随机生成，风格不一、缺乏品牌感。

### 1. 输入输出

- **输入**：文章主题 + 关键词 + 标题
- **输出**：一段供 `image-generate` 使用的封面图 prompt

### 2. 封面类型

| 类型 | 说明 |
|------|------|
| 文字海报 | 纯字 + 简洁背景，大字标题 |
| 情绪风景 | 自然场景 + 小字标题 |
| 产品陈列 | 主体居中 + 标签 |
| 对比拼贴 | 左右分屏 + 两行字 |
| 极简抽象 | 几何图形 + 留白 |

### 3. 六维控制参数

- **色板**：从主题推导配色方案：科技蓝 / 健康绿 / 教育暖 / 严肃灰 / 活力橙 / 专业深蓝
- **渲染**：平涂 / 渐变 / 实拍 / 插画 / 3D
- **文字布局**：居中 / 左对齐 / 居下 / 斜排
- **情绪**：专业 / 轻松 / 权威 / 冲击 / 温暖 / 好奇
- **字体风格**：黑体 / 衬线 / 手写 / 卡通
- **品牌一致性**：同一系列保持配色 + 字体统一

### 4. 流程

1. **分析主题**：从用户输入提取主题、关键词、标题。完成标志：输出三要素，缺任一则追问。
2. **确定类型 + 六维参数**：按封面类型表逐一匹配触发特征——有产品→产品陈列，有对比→对比拼贴，纯文字→文字海报，重情绪→情绪风景，无具体对象→极简抽象。匹配后逐一确定色板/渲染/文字布局/情绪/字体/品牌一致性。
3. **生成自然语言描述**：写 400-600 字的封面图 prompt（400 字覆盖核心构图+配色+文字，600 字追加细节元素+氛围描述——低于 400 信息不足，高于 600 图像生成模型易忽略后半段），覆盖构图、配色、文字、风格全部维度。
4. **调用 prompt-refine 打磨**：传入原始 prompt，获取优化版本。
5. **输出给 image-generate**：将打磨后的 prompt 输出为可直接使用的文本。

### 5. 封面规范

- 公众号封面：900 × 383
- 小红书封面：1080 × 1440

### 6. 边界

- 不直接出图（交给 `image-generate`）
- 不包含平台特定模板
- 不做动态/视频封面（仅静态图）
- 不负责文案撰写（只做视觉描述）

### 失败路径

| 情况 | 信号 | 处理 |
|------|------|------|
| 输入缺少主题/标题 | 用户只说"做个封面"无内容 | 追问："请提供文章主题、关键词和标题" |
| 用户要求直接出图 | "帮我生成一张图" | 拒绝并指引："本 skill 生成封面图 prompt，出图请使用 image-generate" |
| 无法确定封面类型 | 主题太宽泛，无明确方向 | 列出 5 种类型让用户选择，或基于主题推荐最匹配的 1 种并说明理由 |
| prompt-refine 返回失败 | 超时或返回异常 | 跳过打磨，直接输出原始 prompt，标注"未经 prompt-refine 打磨" |
| 用户要求动态/视频封面 | "做个动图""要视频封面" | 拒绝并说明："本 skill 只生成静态封面图 prompt" |

### 7. 完整示例

**主题**："AI编程工具测评"
**关键词**：Cursor、Copilot、效率、代码生成
**标题**：《2025 AI编程工具横评：谁是最强辅助？》

**六维参数**：
- 类型：对比拼贴
- 色板：科技蓝 + 活力橙（左蓝右橙，体现新旧对比）
- 渲染：插画风格
- 文字布局：左对齐（左半屏放标题）
- 情绪：权威 + 冲击
- 字体风格：黑体（科技感）

**封面图 prompt**（约500字）：

> 一张横版对比拼贴风格封面图，左右分屏构图，左侧深蓝色科技背景，右侧活力橙色背景，中间一条斜线分割。左侧用白色粗黑体大字竖排显示"Cursor"，右上角小标签"vs"，右侧用白色粗黑体大字竖排显示"Copilot"。背景两侧各有一组抽象几何代码块元素（大括号、尖括号、线条节点），插画风格，平涂上色，略带微渐变。整体情绪权威且具有冲击力，适合科技评测类文章。标题文字"2025 AI编程工具横评"以白色无衬线字体居下排列，副标题"谁是最强辅助？"缩小一行紧跟。画面干净、对比强烈、阅读层级清晰。分辨率适配公众号封面 900×383 比例，文字大小保证缩略图可读。
