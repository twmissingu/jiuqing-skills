# jiuqing-title-recommend

为内容生成高分标题候选，自动评分后输出 Top 3。嵌入自动化工作流，全程无需用户介入。

## 输入

| 参数 | 类型 | 必传 | 说明 |
|------|------|------|------|
| filePath | string | 否 | 文档文件路径，agent 自行读取 |
| inlineContent | string | 否 | 内联文本内容，与 filePath 二选一 |
| platform | string | 是 | 目标平台：公众号 / 小红书 / B站 / 抖音 / 通用 |

filePath 和 inlineContent 必须提供其一。

## 流程

1. **读取内容**：读 filePath 或 inlineContent，提取核心主题、关键论点、数据点、目标受众。不向用户追问。
2. **加载参考文件**（共 2 个）：
   - `references/methodology.md` — 10 公式 + 5 维度评分标准（必读）
   - `references/platform/{platform}.md` — 平台特征 + 专属公式（按需读取）
   - 文件映射：公众号→wechat.md，小红书→xiaohongshu.md，B站→bilibili.md，抖音→douyin.md，通用→tongyong.md，其他→tongyong.md
3. **生成候选**：10 通用公式 × 3 变体（利益型/冲突型/好奇型，定义见 methodology.md）= 30 候选，+ 平台专属公式全部生成（3–4 个公式 × 1 变体 = 3–4 额外候选）。总计 33–34 候选。
4. **评分**：按 methodology.md 的 5 维度（清晰度/点击理由/具体性/好奇心/可信度）对所有候选打分（1–5，总分 25），应用平台文件中的评分调整规则。
5. **排序**：总分降序排列，平分按 methodology.md 打破规则处理，取 Top 3。
6. **输出**：按下方格式输出，不附理由。

## 输出格式

若有降级标注（如「无专属规范」「内容主题不明确」「专属公式加载失败」），先输出标注行，再输出标题表格：

```markdown
> [标注信息]

## 标题推荐 · [平台名]

| 排名 | 标题 | 清晰度 | 点击理由 | 具体性 | 好奇心 | 可信度 | 总分 |
|------|------|--------|----------|--------|--------|--------|------|
| 1 | [标题A] | 5 | 5 | 4 | 4 | 5 | 23 |
| 2 | [标题B] | 4 | 5 | 5 | 3 | 5 | 22 |
| 3 | [标题C] | 4 | 4 | 4 | 5 | 4 | 21 |
```

## 边界

- 不编造源文中不存在的数据、案例或引用——标题中所有数字必须可溯源到原文
- 不自动发布，标题仅为候选
- 不做 A/B 测试，不承诺效果（"10万+""爆款"等）
- 不做 SEO / meta 描述 / 关键词标签
- 无反馈循环，单次执行即结束

## 失败路径

| 场景 | 处理 |
|------|------|
| 文件路径无效或不存在 | 终止，输出「无法读取文件：[路径]」 |
| 内容 < 50 字 | 终止，输出「内容过短（[N]字），不足以生成标题」 |
| 平台不在已知列表 | 降级用 tongyong.md，标注「无专属规范，按通用标准生成」 |
| methodology.md 缺失 | 终止，输出「skill 安装不完整，请检查 references/methodology.md」 |
| 平台文件缺失 | 降级仅用通用公式，标注「[平台]专属公式加载失败」 |
| 主题不明确 | 泛化生成，标注「内容主题不明确，标题基于全文泛化生成」 |
| 全部候选 < 12 分 | 仍输出 Top 3，标注「所有候选评分偏低，建议人工复核」 |
| 标题编造源文不存在的数据 | 禁止，标题中所有数字/案例必须可溯源到原文，违反则降分至可信度 1 |
| 内容仅有标题无正文 | 终止，输出「内容仅为标题，无正文内容，不足以生成标题」 |
| 内容语言与平台不匹配 | 降级生成，标注「内容语言与平台不匹配，标题可能需要翻译」 |
