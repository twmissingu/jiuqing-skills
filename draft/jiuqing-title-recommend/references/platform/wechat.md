# 公众号平台规范

适用于目标平台为「公众号」或「微信公众号」的标题生成。

## 平台特征

- **核心特征**：信息密度高，包含核心关键词，事件紧迫感，强名词，可见风险/代价
- **标题字数**：20–30 字（过短信息密度不够，过长在分享卡片中被截断）
- **读者行为**：从朋友圈/会话分享点入，扫标题决定是否打开，容忍信息量大的标题
- **调性**：可严肃可活泼，但必须有"料"——空洞标题在公众号生态中 CTR 极低

## 评分调整

| 条件 | 调整 |
|------|------|
| 标题 < 20 字 | 清晰度 -1（信息密度不足） |
| 包含具体数字/金额/时间 | 具体性 +1（上限 5） |
| 包含强名词（人名/公司/产品/政策） | 点击理由 +1（上限 5） |
| 标题 > 30 字 | 清晰度 -1（分享卡片截断） |

## 专属公式

### 1. 事件紧迫式
模板：「刚刚，[主体][动作]：[最大变化/结果]」
示例：「刚刚，OpenAI发布新模型：代码能力直接翻倍」

### 2. 反常识验证式
模板：「我花了 [时间/金钱] 验证了一个 [常识]，结果是 [反转]」
示例：「我花了3个月验证了一个写作常识，结果完全相反」

### 3. 清单紧迫式
模板：「[时间] 前必须知道的 [N] 件事，少一件都吃亏」
示例：「2026年前必须知道的5个AI趋势，少一个都掉队」

### 4. 深度拆解式
模板：「[事件/产品] 背后，藏着 [N] 层逻辑」
示例：「Sora发布的背后，藏着3层你没想到的逻辑」
