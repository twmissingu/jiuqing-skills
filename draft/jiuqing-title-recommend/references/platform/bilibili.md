# B站平台规范

适用于目标平台为「B站」或「Bilibili」的标题生成。

## 平台特征

- **核心特征**：悬念/好奇心缺口，hook-based，标题本身就是钩子
- **标题字数**：20–25 字（B站标题栏可显示较长内容）
- **读者行为**：推荐流+搜索双通道，标题需要在技术/知识类受众中建立信任
- **调性**：可以硬核，可以趣味，但必须有"信息增量"的承诺

## 评分调整

| 条件 | 调整 |
|------|------|
| 包含悬念/好奇心缺口 | 好奇心 +1（上限 5） |
| 包含测试/实测/拆解等验证词 | 可信度 +1（上限 5） |
| 标题 < 18 字 | 清晰度 -1（信息不足） |
| 标题 > 28 字 | 清晰度 -1（信息过载） |

## 专属公式

### 1. 实测验证式
模板：「我花了 [时间] 实测了 [N] 个 [品类]，只有 [X] 个值得买」
示例：「我花了一周实测了8个AI写作工具，只有2个真正能用」

### 2. 硬核拆解式
模板：「[技术/产品] 的底层逻辑，用 [N] 分钟讲透」
示例：「RAG的底层逻辑，用10分钟讲透」

### 3. 争议表态式
模板：「为什么我说 [观点]，可能和你想的不一样」
示例：「为什么我说提示词工程正在消亡，可能和你想的不一样」

### 4. 翻车复盘式
模板：「[失败案例] 踩过的坑，我帮你总结了 [N] 条」
示例：「我做公众号踩过的坑，帮你总结了7条血泪教训」
