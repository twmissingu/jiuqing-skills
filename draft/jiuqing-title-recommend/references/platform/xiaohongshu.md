# 小红书平台规范

适用于目标平台为「小红书」的标题生成。

## 平台特征

- **核心特征**：emoji + 短标题，口语化强，像在跟朋友说话
- **标题字数**：15–20 字（小红书标题显示区域有限，超长被截断）
- **读者行为**：信息流中快速滑动，3 秒内决定是否点击，依赖视觉+情绪
- **调性**：轻松、真实、有共鸣感。避免"官方腔"和"营销腔"

## 评分调整

| 条件 | 调整 |
|------|------|
| 标题 > 20 字 | 清晰度 -1（信息流截断） |
| 使用 emoji（1-3 个） | 点击理由 +1（上限 5） |
| 口语化表达（"真的绝了""姐妹""救命"） | 点击理由 +1（上限 5） |
| 纯书面语/官方腔 | 点击理由 -1 |

## 专属公式

### 1. 种草安利式
模板：「[物品/方法] 真的绝了！[具体效果]」
示例：「这个AI写作工具真的绝了！效率直接翻3倍」

### 2. 避雷吐槽式
模板：「求求你们别再 [错误做法] 了！[正确做法] 它不香吗」
示例：「求求你们别再用'震惊体'了！这个方法它不香吗」

### 3. 身份共鸣式
模板：「[身份标签] 看过来！[好处/发现]」
示例：「自媒体姐妹看过来！这个标题公式真的有用」

### 4. 对比种草式
模板：「用了 [A] 再用 [B]，回不去了」
示例：「用了Claude再用GPT，真的回不去了」
