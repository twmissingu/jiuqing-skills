# jiuqing-roles-debate

让多个角色对一份产出物展开**多轮独立评审 + 主持人收敛**的讨论，逐轮逼近共识，最终把共识结论落到产出物上。

核心区别于单次并行评审：这是**迭代**的——每轮把上一轮的分歧作为新议题再讨论，直到不再产生新的实质分歧。

## 何时不该用

- 只是想一次性听几个视角的意见、不需要迭代收敛 → 用一次并行评审即可，别套多轮
- 一句话就能改的小问题、或目标根本未定 → 先把需求理清，别用讨论凑数

## 流程

### 步骤 1：读取目标与自判断范围

读取用户指定的产出物（文档 / 代码 / 设计稿 / 项目说明等）。内容过大（如超 5000 字或多文件）时，先问用户是全量还是聚焦某部分。

读完后**自行判断**并形成一份"讨论范围提案"：

- **讨论目标** —— 这次讨论要解决什么？（评审找问题 / 推演方案可行性 / 在几个选项里抉择 / 补全缺口）
- **候选角色** —— 根据产出物类型和目标，提名合适的角色（不要套固定清单，按现场选；例如 API 设计可能要后端、前端、安全、SRE；商业方案可能要产品、运营、财务、法务）
- **纳入范围** —— 哪些部分参与讨论，哪些明确排除

### 步骤 2：用户确认范围

把上述提案一次性列给用户确认，允许增删角色、调整目标与范围。确认后再开讨。**不要跳过这步自作主张开讨。**

### 步骤 3：多轮讨论（核心）

每一轮按下面的"独立发言 → 主持人收敛"循环执行：

1. **并行独立发言** —— 用 Agent 工具为**每个角色各起一个子代理**并行执行，让它们互不可见地独立发言。隔离上下文是为了避免角色在同一对话里互相迎合、趋同。每个子代理 prompt 必须自包含，结构见下方"角色发言 prompt 模板"。从第 2 轮起，prompt 中附上"主持人本轮议题"（即上一轮未收敛的分歧点），让发言聚焦争议而非重复已有共识。

2. **主持人收敛** —— 收齐本轮所有角色发言后，由主 agent（主持人）做三件事：
   - **归并共识**：≥2 个角色一致认可的点，登记进"共识区"，后续不再重复讨论
   - **提炼分歧**：角色之间冲突或单方提出待议的点，登记进"分歧区"，作为下一轮议题
   - **判断收敛**：见步骤 4

每轮结束向用户简报一行：本轮新增共识 N 条、未决分歧 M 条、当前第 K 轮。

### 步骤 4：收敛判定

满足任一即停止讨论：

- **无新增实质分歧** —— 连续一轮（含）以上没有冒出新的实质性分歧，且存量分歧要么已被裁决、要么角色立场固化不再变化（此时如实记录为"保留分歧"交用户裁决）。这是首要的收敛信号。
- **触顶 20 轮** —— 20 轮是底线上限（经验值：多数有效讨论在 3-8 轮内收敛；超过 12 轮通常意味着议题本身有根本分歧，继续讨论收益递减。20 轮留足缓冲但不会空耗过多 token）。即便仍有分歧，到 20 轮也停，把未决项整理为"保留分歧"。
- **≥2 个角色达成共识** —— 当 ≥2 个角色对某个议题给出一致判断（同意或等价立场），该议题可直接归入共识区，不再重复讨论。这是加速收敛的信号——不需要全体一致，多数一致即可推进。

收敛不等于零分歧——固化的对立观点应保留双方立场，交用户裁决，而非强行统一。

### 步骤 5：产出共识结论与行动清单

讨论收敛后，整理输出：

- **共识结论** —— 讨论达成的一致判断，按主题归类
- **保留分歧** —— 未统一的点，列出各方立场与理由，标注待用户裁决
- **行动清单** —— 把共识转成对产出物的具体改动，每条含：序号、改动描述、改动类型（新增/修改/删除）、来源角色、预估工作量。拒绝"建议加强 XX"之类空话，每条必须可执行。

### 步骤 6：确认后落地

把行动清单交用户逐条确认（全部同意 / 指定序号 / 跳过某条 / 调整某条）。**先出方案经确认，再动产出物。** 确认后把改动攒成一批，执行前展示变更预览（diff 或 old→new），用户确认后再写入。

## 角色发言 prompt 模板

每个角色子代理的 prompt 按此结构自包含填写：

```
你是【角色名】，正在参与一场对某产出物的多轮评审讨论。

## 你的身份与立场
[一句话身份 + 你最在意什么]

## 你的关注维度
[3-5 条该角色特有的审查维度]

## 你的评判红线
[触发即判高危的硬条件，2-3 条]

## 产出物内容
[完整内容或聚焦部分]

## 主持人本轮议题（第 2 轮起）
[上一轮未收敛的分歧点；首轮省略此节]

## 输出要求
1. 整体定性：一句话（可行 / 需大改 / 方向有误）
2. 具体问题：按严重程度排序，每条须引用产出物原文（> 引用），说明问题本质，标注 🔴高/🟡中/🟢低
3. 改进建议：与问题一一对应，给具体方案，标注改动类型与工作量
4. 对本轮议题的立场（第 2 轮起）：对每个议题表态——同意他人观点 / 坚持原立场（给新论据）/ 提出折中
5. 若产出物未涉及你的关注领域，直说"未涉及本角色领域"并跳过，不硬凑
```

## 注意事项

- 角色发言必须引用原文，不做无据批评
- 隔离独立是关键：永远不要把一个角色的发言塞进另一个角色的发言上下文，只能由主持人在收敛环节统一消化
- 议题逐轮收窄：共识项移出讨论，只把未决分歧带进下一轮
- 不替用户裁决保留分歧，呈清各方立场即可

## 失败路径

| 场景 | 信号 | 处理 |
|------|------|------|
| 角色发言质量退化 | 后续轮次角色重复相同观点无新论据 | 主持人指出退化，要求换角度或结束该角色参与 |
| 讨论跑题 | 发言偏离产出物本身 | 主持人拉回议题，明确"下一轮只讨论 X" |
| 用户要求跳过讨论直接给结论 | "别讨论了直接改" | 解释多轮讨论的价值，如用户仍坚持则按单次评审处理 |
